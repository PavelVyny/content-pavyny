    **File 1: SKILL.md** (per D-07: `.claude/skills/devlog-scriptwriter/SKILL.md`, under 500 lines per SKIL-01)

    Use the official frontmatter from research:
    ```yaml
    ---
    name: devlog-scriptwriter
    description: >
      Generates natural-sounding YouTube Shorts devlog scripts with anti-slop
      scoring and brand voice matching. Handles ideation (topic angles from dev
      progress), script generation (dual-track visual + voiceover), and metrics
      analysis (pattern detection from performance data). Use when writing video
      scripts, brainstorming devlog ideas, analyzing video performance, or when
      the user mentions scripts, devlog, YouTube, shorts, video ideas, or hook.
    ---
    ```

    Body structure -- three modes (from research Multi-Mode Skill Pattern):

    **Mode: Ideation** -- When asked for ideas, topics, or angles:
    1. Read metrics-log.md for performance patterns
    2. Ask about this week's dev progress
    3. Generate 5-7 angles using formats from video-formats.md
    4. Tag each angle with format name and visual potential
    5. Apply one-idea-per-video rule

    **Mode: Script Generation** -- When asked to write a script:
    1. Read brand-voice.md for voice profile
    2. Read anti-slop-rules.md for constraints
    3. Read video-formats.md for the chosen format template
    4. Generate script in dual-track format:
       - Left column: VISUAL (what viewer sees -- screen recording description)
       - Right column: VOICEOVER (what Pavlo says)
    5. Apply hook structure: Pre-hook visual (0-1s) -> Question/statement (1-3s) -> Deliver (3s+)
    6. Generate 2-3 hook variants
    7. Generate 3 title options + thumbnail frame concept
    8. Self-score against anti-slop rubric (must be 35+/50)
    9. If score < 35, rewrite violations using brand voice and re-score
    10. One-idea enforcement check: if script contains multiple distinct topics, split into separate script suggestions

    **Mode: Analysis** -- When asked to analyze metrics or performance:
    1. Read metrics-log.md
    2. Compare formats, hooks, topics, durations
    3. Generate actionable rules for next generation cycle

    **Additional Resources section** (using official markdown link pattern):
    ```markdown
    ## Additional resources

    - For video format templates, see [video-formats.md](references/video-formats.md)
    ```

    - Anti-slop is priority #1
    - Specificity = credibility ("I rewrote collision detection 3 times" not "I worked on physics")
    - One Short = one idea
    - Visuals drive, voice follows
    - Scripts must be easy to pronounce for non-native English speaker
    - Forward slashes in all paths (even on Windows)

    **File 2: anti-slop-rules.md** (per ECOS-04: integrate best-of-ecosystem)

    Structure from research Anti-Slop Rules File Structure:

    - **Scoring Rubric**: 5 dimensions from stop-slop (Directness, Rhythm, Trust, Authenticity, Density) with score descriptions. Each 1-10, total 35+/50.
    - **Banned Phrases (60+)**: Combine from ecosystem audit findings:
      - stop-slop core rules (filler phrases, formulaic structures)
      - humanizer language patterns (AI vocabulary: "Additionally", "testament", "landscape", etc.)
      - slop-radar buzzwords (if found in audit)
      - anti-slop-writing rules (if found in audit)
      - CLAUDE.md banned list (journey, dive into, game-changer, incredible, leverage, etc.)
      Organize by category: AI vocabulary, filler phrases, hype language, YouTuber cliches, corporate speak.
    - **Structural Rules**: From stop-slop 8 rules + humanizer structural patterns, filtered for script relevance:
      1. Cut filler phrases
      2. Break formulaic structures
      3. Use active voice
      4. Be specific (numbers, names, timeframes)
      5. Put viewer in the room
      6. Vary rhythm
      7. Trust viewer intelligence
      8. Cut quotables (no "tweetable" phrasing)
    - **Detection Patterns**: Top patterns from humanizer's 24 categories relevant to scripts:
      - Significance inflation
      - Promotional language
      - Rule of three
      - Synonym cycling
      - Sycophantic tone
      - Filler phrases
      - Generic conclusions

    **File 3: video-formats.md**

    7 proven format templates from CLAUDE.md:
    1. **The Bug** -- "This bug broke my game for 3 days" (problem -> struggle -> fix -> result)
    2. **The Satisfaction** -- satisfying mechanic/sound/visual (setup -> reveal -> slow-mo/replay)
    3. **Before/After** -- transformation (old state -> what changed -> new state)
    4. **The Decision** -- "I almost deleted this feature" (context -> dilemma -> choice -> outcome)
    5. **The Trick** -- "One line of code changed everything" (problem -> insight -> solution -> proof)
    6. **The Fail** -- "A week of work in the trash" (investment -> failure -> lesson/pivot)
    7. **The Number** -- "47 iterations to get this right" (scale -> process -> result)

    For each format include:
    - Hook formula: Pre-hook visual + opening line template
    - Structure: beat-by-beat outline (3-5 beats)
    - Visual suggestions: what to show at each beat
    - Duration guidance: typical 30-47 seconds
    - Example: reference a real Pavlo video that uses this format (where applicable)
