  <action>
Create `web/scripts/migrate-sqlite-to-supabase.ts` — a one-shot migration script.

**Read side (SQLite):** Install `better-sqlite3` and `@types/better-sqlite3` as devDependencies (temporary, removed in Task 2). Open `web/data/scripts.db` read-only with better-sqlite3. Read all rows from each table.

**Timestamp conversion (per D-01):** For each timestamp column (scripts.createdAt, scripts.updatedAt, videos.publishedAt, videos.createdAt, videos.updatedAt, videoMetrics.lastSyncedAt), auto-detect epoch format:
- If value is null/undefined: pass through as null
- If value is a string that parses as ISO date: use `new Date(value)`
- If value is a number > 1e12: treat as milliseconds, use `new Date(value)`
- If value is a number > 0 and <= 1e12: treat as seconds, use `new Date(value * 1000)`
- Log conversion decisions per column for transparency

**Write side (Supabase):** Use postgres-js with `prepare: false` connected via DATABASE_URL from env. Write data using raw SQL (not Drizzle ORM) to allow explicit column mapping and IDENTITY INSERT behavior.

**Table order (FK constraints):** scripts -> beats -> videos -> videoMetrics

**For each table:**
1. Read all rows from SQLite
2. Convert timestamps using the auto-detect logic above
3. Parse JSON string columns into objects (hooks, titles, antiSlopScore, retentionCurve) — SQLite stores JSON as text strings
4. Insert into PostgreSQL with explicit column names including the `id` column (to preserve PKs)
5. After all inserts for a table, reset the serial sequence:
   ```sql
   SELECT setval(pg_get_serial_sequence('table_name', 'id'), (SELECT COALESCE(MAX(id), 0) FROM table_name));
   ```

**Output (per D-02):** Print row counts per table after migration. Format:
```
Migration complete:
  scripts: N rows
  beats: N rows
  videos: N rows
  video_metrics: N rows
```

**Wrap entire migration in a transaction** — if any table fails, roll back everything.

**Column mapping (SQLite snake_case to PostgreSQL):**
- scripts: id, title, format, status, hooks, selected_hook, titles, thumbnail, duration_estimate, anti_slop_score, dev_context, created_at, updated_at
- beats: id, script_id, "order" (quoted — reserved word in PG), visual, voiceover, duration
- videos: id, youtube_id, title, description, thumbnail_url, published_at, channel_title, script_id, created_at, updated_at
- video_metrics: id, video_id, views, engaged_views, likes, comments, shares, subscribers_gained, subscribers_lost, average_view_percentage, average_view_duration, retention_curve, last_synced_at

**Script header:**
```typescript
import "dotenv/config";
import Database from "better-sqlite3";
import postgres from "postgres";
import path from "path";
```

Run with: `npx tsx scripts/migrate-sqlite-to-supabase.ts`
