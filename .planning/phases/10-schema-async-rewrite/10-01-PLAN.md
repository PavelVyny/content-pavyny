# Phase 10 Plan 01: Schema & Async Rewrite

## Objective

Replace local SQLite (better-sqlite3) with Supabase PostgreSQL. Rewrite Drizzle schema from sqliteTable to pgTable, swap the connection module to postgres-js, and convert all synchronous DB call sites to async/await. The app must be functionally identical but running against a remote Supabase database.

## Context

- @.planning/phases/10-schema-async-rewrite/10-CONTEXT.md (decisions D-01 through D-10)
- @src/lib/db/schema.ts (current SQLite schema)
- @src/lib/db/index.ts (current better-sqlite3 connection)
- @drizzle.config.ts (current Drizzle config)

## Tasks

### Task 1: Install postgres-js and update dependencies
type="auto"

**Do:**
1. Install `postgres` (postgres-js) package in web/
2. Remove `better-sqlite3` and `@types/better-sqlite3` from dependencies
3. Add `drizzle-orm/postgres-js` import support
4. Add `@dotenvx/dotenvx` or ensure .env.local is loaded by Next.js (it is by default)

**Done when:**
- `postgres` is in package.json dependencies
- `better-sqlite3` and `@types/better-sqlite3` are removed from package.json
- `npm install` succeeds

### Task 2: Rewrite Drizzle schema from SQLite to PostgreSQL
type="auto"

**Do:**
1. Rewrite `src/lib/db/schema.ts`:
   - Replace `sqliteTable` with `pgTable` from `drizzle-orm/pg-core`
   - Replace `integer("id").primaryKey({ autoIncrement: true })` with `serial("id").primaryKey()`
   - Replace `text("col", { mode: "json" }).$type<T>()` with `jsonb("col").$type<T>()`
   - Replace `integer("col", { mode: "timestamp" })` with `timestamp("col")`
   - Replace `.$defaultFn(() => new Date())` with `.defaultNow()`
   - Keep all column names, references, and constraints identical
   - Use `text` for regular text columns (same in both)
   - Use `integer` for plain integer columns (same in both)

**Done when:**
- schema.ts uses pgTable for all 4 tables
- All type imports are from `drizzle-orm/pg-core`
- No SQLite imports remain

### Task 3: Rewrite DB connection module for postgres-js
type="auto"

**Do:**
1. Rewrite `src/lib/db/index.ts`:
   - Import `postgres` from `postgres` and `drizzle` from `drizzle-orm/postgres-js`
   - Read DATABASE_URL from `process.env.DATABASE_URL`
   - Create postgres-js client with `prepare: false` (for Supabase pooler)
   - Create Drizzle instance with schema
   - Export `getDb()` that returns the Drizzle instance (keep same API)
   - Throw clear error if DATABASE_URL is missing
2. Update `drizzle.config.ts`:
   - Change dialect to `"postgresql"`
   - Use DATABASE_URL from env for dbCredentials
3. Create `.env.local.example` with DATABASE_URL placeholder

**Done when:**
- db/index.ts uses postgres-js driver
- drizzle.config.ts uses postgresql dialect
- .env.local.example exists

### Task 4: Convert all server actions to async DB calls
type="auto"

**Do:**
1. In every server action file (generate.ts, editor.ts, library.ts, metrics.ts):
   - Change `const db = getDb()` to `const db = getDb()` (same, but all queries need await)
   - Replace all `.get()` with await (Drizzle postgres returns promises)
   - Replace all `.all()` with await
   - Replace all `.run()` with await
   - Replace `.returning().get()` with await + array access `[0]`
   - Replace `onConflictDoUpdate` SQL references: `sql\`excluded.col\`` stays the same for PostgreSQL
   - Ensure every DB call has `await`

**Done when:**
- No remaining `.get()`, `.all()`, `.run()` terminal methods
- All DB operations are awaited
- All 4 server action files compile without errors

### Task 5: Convert page server components to async DB access
type="auto"

**Do:**
1. `src/app/page.tsx` (Home):
   - Make default export `async function`
   - Add await to all DB queries
2. `src/app/script/[id]/page.tsx` (already async):
   - Add await to DB queries (getDb() calls)
3. `src/app/scripts/page.tsx` (already async via server actions):
   - Verify no direct DB calls need conversion
4. `src/app/analytics/page.tsx` (already async via server actions):
   - Verify no direct DB calls need conversion
5. Update `next.config.ts`:
   - Remove `serverExternalPackages: ["better-sqlite3"]` (no longer needed)

**Done when:**
- All pages compile without errors
- Home page is async
- next.config.ts no longer references better-sqlite3

### Task 6: Push schema to Supabase and verify
type="auto"

**Do:**
1. Run `npx drizzle-kit push` to create tables on Supabase
2. Write a smoke test script `scripts/smoke-test.ts` that:

3. Run the smoke test

**Done when:**
- All 4 tables exist in Supabase
- Smoke test passes all operations
- No SQLite references remain in the codebase (except next.config comment if any)
