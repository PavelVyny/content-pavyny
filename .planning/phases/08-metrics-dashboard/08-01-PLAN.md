Add three new exported functions to the end of youtube-client.ts:

1. `listChannelVideos()` — Uses two-step pattern per RESEARCH Pattern 1:
   - Call `google.youtube("v3").channels.list({ auth: client, part: ["contentDetails"], mine: true })` to get uploads playlist ID from `items[0].contentDetails.relatedPlaylists.uploads`
   - Call `google.youtube("v3").playlistItems.list({ auth: client, part: ["snippet"], playlistId, maxResults: 50, pageToken })` in a loop to get all videos
   - Return `Array<{ youtubeId: string; title: string; description: string; thumbnailUrl: string; publishedAt: string }>`
   - Handle pagination with `nextPageToken` loop
   - If no uploads playlist found, return empty array

2. `getVideoMetrics(videoIds: string[], channelStartDate: string)` — Per RESEARCH Pattern 2:
   - Call `google.youtubeAnalytics("v2").reports.query({ auth: client, ids: "channel==MINE", startDate: channelStartDate, endDate: today (YYYY-MM-DD), dimensions: "video", metrics: "views,likes,comments,shares,subscribersGained,subscribersLost,averageViewPercentage,averageViewDuration,engagedViews", filters: \`video==${videoIds.join(",")}\`, sort: "-views" })`
   - Return `response.data.rows ?? []` (array of arrays: [videoId, views, likes, comments, shares, subsGained, subsLost, avgViewPct, avgViewDuration, engagedViews])
   - If videoIds is empty, return empty array early

3. `getRetentionData(videoId: string, channelStartDate: string)` — Per RESEARCH Pattern 3:
   - Call `google.youtubeAnalytics("v2").reports.query({ auth: client, ids: "channel==MINE", startDate: channelStartDate, endDate: today, dimensions: "elapsedVideoTimeRatio", metrics: "audienceWatchRatio,relativeRetentionPerformance", filters: \`video==${videoId};audienceType==ORGANIC\`, maxResults: 200 })`
   - Map rows to extract audienceWatchRatio (index 1): `(response.data.rows ?? []).map(row => Number(row[1]))`
   - Return `number[]` (100-point retention curve, values can exceed 1.0 for rewatches)

All three use `getOAuth2Client()` for auth. Use `google.youtube("v3")` for Data API and `google.youtubeAnalytics("v2")` for Analytics API. Both are available from the already-installed `googleapis` package.

Do NOT use `search.list` (100 quota units). Use `playlistItems.list` (1 unit).

**Create web/src/app/actions/metrics.ts** as a new "use server" file with these exports:

1. `syncMetrics()` — Full sync flow per RESEARCH Code Examples:
   - Call `listChannelVideos()` to discover videos
   - Upsert each video into `videos` table using `db.insert(videos).values({...}).onConflictDoUpdate({ target: videos.youtubeId, set: { title, updatedAt: new Date() } }).run()`
   - Call `getVideoMetrics(videoIds, "2024-01-01")` for batch basic metrics
   - For each metrics row: find the DB video by youtubeId, call `getRetentionData(youtubeId, "2024-01-01")`, then upsert into `videoMetrics` using `onConflictDoUpdate({ target: videoMetrics.videoId, set: { views, likes, comments, shares, subscribersGained, subscribersLost, averageViewPercentage, averageViewDuration, engagedViews, retentionCurve, lastSyncedAt: new Date() } })`
   - Call `revalidatePath("/scripts")` at the end
   - Return `{ success: boolean; videoCount: number; error?: string }`
   - Wrap in try/catch, return error message on failure

2. `getScriptsWithMetrics()` — Query for scripts page:
   - Left join scripts -> videos (on `videos.scriptId = scripts.id`) -> videoMetrics (on `videoMetrics.videoId = videos.id`)
   - Filter out `scripts.status = "generating"`
   - Order by `scripts.createdAt` desc
   - Return `ScriptWithVideo[]`

3. `getLastSyncTime()` — For staleness indicator:
   - Query `videoMetrics` for the MAX `lastSyncedAt`
   - Return `Date | null`

4. `linkVideo(scriptId: number, videoId: number)` — For LINK-01:
   - Update `videos` table: set `scriptId` on the video row where `id = videoId`
   - Call `revalidatePath("/scripts")` and `revalidatePath(\`/script/${scriptId}\`)`
   - Return `{ success: boolean }`

5. `unlinkVideo(videoId: number)` — For LINK-02:
   - Update `videos` table: set `scriptId = null` where `id = videoId`
   - Call `revalidatePath("/scripts")` and `revalidatePath("/script/*")` (broad revalidation since we don't know the old scriptId)
   - Return `{ success: boolean }`

6. `getUnlinkedVideos()` — For video linking dropdown:
   - Query `videos` where `scriptId IS NULL`
   - Return `VideoData[]`

7. `getVideoForScript(scriptId: number)` — For editor page:
   - Query `videos` with join to `videoMetrics` where `videos.scriptId = scriptId`
   - Return `{ video: VideoData; metrics: VideoMetricsData } | null`

Import from drizzle-orm: `eq, desc, ne, isNull, sql`. Import schema tables: `scripts, videos, videoMetrics`. Import youtube-client functions: `listChannelVideos, getVideoMetrics, getRetentionData`.
