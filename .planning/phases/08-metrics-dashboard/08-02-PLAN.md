**Create web/src/components/sync-button.tsx** — Per D-05, D-06, D-07, D-08:
- "use client" component
- Props: `{ lastSyncedAt: Date | null }` (serialized as string from server, parse on client)
- State: `syncing: boolean`, `progress: { current: number; total: number } | null`, `result: { videoCount: number; error?: string } | null`
- Staleness computation (per D-07):
  - `null` lastSyncedAt -> red, "Never synced"
  - `< 1 hour` -> green badge, relative time via `Intl.RelativeTimeFormat("en", { numeric: "auto" })`
  - `< 24 hours` -> yellow badge (bg-yellow-100 text-yellow-800)
  - `>= 24 hours` -> red badge (bg-red-100 text-red-800)
- "Sync Now" button: uses shadcn/ui Button. Disabled while `syncing` is true (per D-08).
- On click handler — two-phase sync per D-06:
  1. Set syncing=true, progress={ current: 0, total: 0 }
  2. Call `discoverVideos()` server action. On success, set progress.total to videos.length. Button text updates to `Syncing 0/${total} videos...`
  3. Loop through returned videos array: for each video, call `syncSingleVideo(video.youtubeId)`. After each completes, increment progress.current. Button text updates to `Syncing ${current}/${total} videos...` (e.g., "Syncing 3/6 videos...")
  4. After all complete: set syncing=false, set result with videoCount=total
  5. If discoverVideos or any syncSingleVideo fails: set syncing=false, show error
- IMPORTANT per D-06: The button text MUST show count-based progress "Syncing 3/6 videos..." — NOT just "Syncing..." or a spinner. This is a locked user decision.
- On error: show brief error message in red text below the button, disappears after 5 seconds (per D-08).
- On success: show "Synced N videos" in green text briefly (3 seconds).
- Layout: horizontal flex with staleness badge on left, sync button on right.
