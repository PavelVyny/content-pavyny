    Create `src/app/actions/generate.ts` as a Server Action ("use server").

    1. Export async function `generateNewScript(formData: FormData)`:
       - Extract `format` (string) and `devContext` (string) from FormData
       - Validate: both required, devContext minimum 10 characters. Return `{ error: string }` on validation failure.

    2. Insert a placeholder script row with `status: "generating"` and `title: "Generating..."` so the UI can show progress:
       ```typescript
       const db = getDb();
       const [script] = db.insert(scripts).values({
         title: "Generating...",
         format,
         status: "generating",
         devContext,
       }).returning();
       ```

    3. Call `generateScript(format, devContext)` in a try/catch:
       - On success: update the script row with all generated data (title from `output.titles[0]`, hooks, titles, thumbnail, durationEstimate, antiSlopScore), set status to "draft"
       - Insert beats as separate rows in the beats table: map `output.beats` to `{ scriptId: script.id, order: index + 1, visual, voiceover, duration }`
       - Call `revalidatePath("/")` to refresh the page
       - Return `{ success: true, scriptId: script.id }`

    4. On error: update the placeholder script's title to "Generation failed", status to "draft", return `{ error: errorMessage }`.

    5. Import `eq` from `drizzle-orm` for the update where clause.

    6. Export async function `regenerateScript(scriptId: number, format: string, devContext: string)`:
       - Delete existing beats for this script: `db.delete(beats).where(eq(beats.scriptId, scriptId)).run()`
       - Set script status to "generating"
       - Call `generateScript(format, devContext)` same as above
       - Update script row with new data, insert new beats
       - Per GENR-03: allows re-generation with same or modified input
       - Return `{ success: true, scriptId }` or `{ error: string }`
  </action>
