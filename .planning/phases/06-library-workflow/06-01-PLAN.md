Create `web/src/app/actions/library.ts` with three server actions following the pattern in editor.ts:

1. `getAllScripts()` — Returns all scripts (without beats) ordered by `createdAt` DESC (per D-04). Select all columns from the `scripts` table. Cast JSON fields (hooks, titles, antiSlopScore) to their TypeScript types. Filter out scripts with status "generating" (those are in-progress). Return typed `Script[]`.

2. `updateScriptStatus(scriptId: number, status: "draft" | "ready" | "recorded")` — Updates the script's status and updatedAt fields. Returns `{ success: boolean }`. Per D-03, this powers the inline dropdown. Do NOT accept "generating" as a valid status value.

3. `getVoiceoverText(scriptId: number)` — Fetches all beats for the given script ordered by `beats.order` ASC, plus the selected hook variant's voiceover from the script's hooks array. Returns `{ success: boolean, text?: string }`. The text format per D-06: selected hook voiceover first (if any), then each beat's voiceover on its own line, separated by blank lines between beats. No visual text, no labels, no formatting — just clean voiceover text ready for recording reference.
